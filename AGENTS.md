## Stack
- Zsh dotfiles repo — shell environment config, NOT an application. No package manager, no build step, no test runner.
- Managed toolchain via `mise` (Node 24.18.0 / Python 3.11.15 / Rust 1.97.1 pins). Homebrew for system packages.

## Files
- `zshenv` — sourced for EVERY zsh (login + interactive + non-interactive `zsh -c`). Keep minimal: only PATH/exports needed in non-interactive contexts (e.g. cargo).
- `zprofile` — login-shell only. Terminal settings, exports, PATH bootstrap. Activates `mise` AFTER brew/.local so shims win.
- `zshrc` — interactive non-login. Prompt logic, aliases, interactive functions.
- `mise.toml` / `.mise.toml` — toolchain pins + settings (`experimental = true`).
- `Dracula.terminal` — macOS Terminal theme profile.
- `README.md` — documents the split + shared config rules.

## Commands
- Apply changes: source the file, or open a new shell. No build/install.
- Edit then `source ~/.zprofile` (or `~/.zshrc`) to test — or just restart the terminal.
- mise pins live in `~/.config/mise/config.toml`; repo `.mise.toml` is trusted + overrides globals.

## Gotchas / Forbidden
- NEVER commit username-specific absolute paths (`/Users/amf/...`, `/Users/your-name/...`). Use `$HOME` / `$DEV_DIR` generics.
- Keep machine-specific hosts/usernames as env vars (e.g. `PI_SSH_TARGET`) — edit shared files directly.
- PATH order matters in `zprofile`: mise activated AFTER brew/.local so shims shadow them. Don't reorder blindly.
- `zshenv` must stay lean — anything heavy breaks non-interactive `zsh -c` (used by build tools/scripts).
- Don't `bun install` / `npm install` / `pip install` — there is no app dependency manifest here.

## Conventions
- PATH exports + machine bootstrap → `zprofile`.
- Prompt/aliases/interactive functions → `zshrc`.
- Shared/portable rules → keep in these three files, documented in README.
